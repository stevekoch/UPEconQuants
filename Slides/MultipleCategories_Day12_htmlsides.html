Some Background

Applied research questions

  1. Does education influence employment prospects?

  2. Does the size of the household reduce access to:

    1. food?
    2. healthcare?
    3. clean energy?

  3. Which illnesses lead to catastrophic health care expenditure?

  4. Does alcohol/drug use/debt influence mental health?

  5. Does the age of the predidential candidate affect willingness to vote?

What is the common denominator?

  • The outcome of interest is categorical:
    • employment prospects: employed, unemployed, non-participant
    • access to food: inadequate, adequate, more than adequate
    • catastrophic health care expenditure: catastrophic, non-catastrophic
    • mental health: depressed, not depressed
    • voting: cast ballot or not

That means?

  • Categorical outcomes are not Gaussian

  • Errors are synonymous with mis-classification

How should we think about modelling?

  • Two options:
    1. Assume/Pretend from unobserved continuous variable
    2. Use a prediction algorithm
    3. Sometimes, these are the same
  • Train (estimate) the model:
    1. Use all the data, no real evaluation options
    2. Create a training set, evaluate on test set

The data

  1. We will use the LCS data (Statistics South Africa 2017)

  2. We will focus on food adequacy

  3. Our controls will be:

Setup Concepts

Binary model structure

  • A series of observations \((y_i,X_i)\)
  • Machine learning thinks of these as training data
  • \(X_i\) is observed characteristics and

\[ y_i = \text{Food adequacy} = \begin{cases} \text{Below adequate} \\ \text{Adequate} \end{cases} \]

Arbitrary values

  • Often, we impose ‘arbitrary’ numerical values, \(\{0, 1 \}\)
  • Should be intuitive

\[ y_i = \text{Food adequacy} = \begin{cases} 0 & \text{ if Below adequate} \\ 1 & \text{ if Adequate} \end{cases} \]

Multinomial ranked responses

  • Keeping all outcomes
  • One could treat them as ranked
    • bad to good
    • good to bad

\[ y_i = \text{Food adequacy} = \begin{cases} \text{Below adequate} \\ \text{Adequate} \\ \text{More than adequate} \end{cases} \]

More on ranked alternatives

  • It is often preferred that the spacing is both
    • obvious, and
    • constant
  • Here, it is obvious, maybe not constant ‘steps’
  • An example with both?

Multinomial unranked outcomes

  • If the ordering is not obvious, it is better to not order them

\[ y_i = \text{Employment Status} = \begin{cases} \text{Employed} \\ \text{Unemployed} \\ \text{Non-participant in the labour force} \end{cases} \]

Another unranked outcome

\[ y_i = \text{Favourite team} = \begin{cases} \text{Arsenal} \\ \text{Manchester City} \\ \text{Liverpool} \\ \text{Manchester United} \end{cases} \]

Ordered Outcome Models

The Probabilistic Setting

  • \(R_i = \gamma Z_i + u_i\) underpins empirical setup

\[ y_i = \begin{cases} 0, & \text{If } u_i \leq \kappa_1 - \gamma Z_i \\ 1, & \text{If } \kappa_1-\gamma Z_i < u_i \leq \kappa_2 - \gamma Z_i \\ 2, & \text{If } \kappa_2 -\gamma Z_i < u_i \end{cases} \]

  • For example

\[ \text{prob}(y_i=2|Z_i) = \text{prob}(u_i>\kappa_2 - \gamma Z_i) \]

  • Need distributional assumption
  • Empirical modeling depends on it

Dependent Variable Structure

  • Define \(p_{ij} \equiv \text{prob}(y_i=j)\)
  • There are \(J \in \{0, J-1\}\) possible alternatives
  • Introduce a generalisation
    • Assume \(F\) is a cdf
    • \(\kappa_0 = -\infty\)
    • \(\kappa_J=\infty\)

\[ \begin{split} \text{prob}(y_i=j|Z_i) &= \text{prob}\left( \kappa_{j-1} - Z_i' \gamma < u_i \leq \kappa_j - Z_i' \gamma \right) \\ &= F \left(\kappa_j - Z_i' \gamma \right) - F\left( \kappa_{j-1} - Z_i' \gamma \right) \end{split} \]

Density per Observation: Likelihood Contribution

  • Again, define \(p_{ij} \equiv \text{prob}(y_i = j)\)
  • Assume \(J\) alternatives: counted from \(0\) to \(J-1\)

\[ f(y_i,p) = p_{i0}^{y_{i0}} \times p_{i1}^{y_{i1}} \times \cdots p_{iJ-1}^{y_{iJ-1}} = \prod_{j=0}^{J-1} p_{ij}^{y_{ij}} \]

Substitute in the probabilities

  • \(N\) independent observations
  • Possible to allow for dependence, through clusters

\[ \begin{split} f(y,p) &= \prod_{i=1}^N \left(p_{i0}^{y_{i0}} \times p_{i1}^{y_{i1}} \times \cdots p_{iJ-1}^{y_{iJ-1}} \right) \\ f(y,\gamma, \kappa) &= \prod_{i=1}^N \prod_{j=0}^{J-1} \left[ F\left( \kappa_j - Z_i' \gamma \right) - F\left( \kappa_{j-1} - Z_i' \gamma \right) \right]^{y_{ij}} \end{split} \]

Multinomial Logit Models

Dependent Variable Structure

  • Recall \(p_{ij} \equiv \text{prob}(y_i=j)\)
  • There are \(J\) possible alternatives
  • Therefore, introduce generalisation

\[ y_{ij} = \begin{cases} 1, & \text {If alternative } j \text{ is chosen} \\ 0, & \text {Otherwise} \end{cases} \]

  • \(y_i\) vector of zeroes, and a single 1

\[ y_i = \begin{pmatrix} 0 & 1 & 0 \end{pmatrix}' \]

The Multinomial Density per Observation

  • Above information and only three outcomes
  • The multinomial density

\[ f(y_i,p) = p_{i1}^{y_{i1}} \times p_{i2}^{y_{i2}} \times p_{i3}^{y_{i3}} = \prod_{j=1}^3 p_{ij}^{y_{ij}} \]

  • Extended to additional alternatives (\(J\) total)

\[ f(y_i,p) = p_{i0}^{y_{i0}} \times p_{i1}^{y_{i1}} \times \cdots p_{iJ-1}^{y_{iJ-1}} = \prod_{j=0}^{J-1} p_{ij}^{y_{ij}} \]

Likelihood Estimation

  • \(N\) observations assumed independent

\[ f(y,p) = \prod_{i=1}^N \left(p_{i0}^{y_{i0}} \times p_{i1}^{y_{i1}} \times \cdots p_{iJ}^{y_{iJ}} \right) = \prod_{i=1}^N \prod_{j=1}^J p_{ij}^{y_{ij}} \]

  • Formalize log likelihood function

\[ \ell = \ln \mathcal{L} = \sum_{i=1}^N \sum_{j=1}^J y_{ij} \ln p_{ij} \]

  • Probabilities are functions of variables and parameters

Concerns: IIA

  • Let us consider \(p_{i2}\) and \(p_{i3}\)
  • Assume categry 1 is the baseline

\[ \begin{aligned} p_{i2} &= \frac{ \exp (x_i \beta_2)}{1 + \exp (x_i \beta_2) + \exp (x_i \beta_3)} \\ p_{i3} &= \frac{ \exp (x_i \beta_3)}{1 + \exp (x_i \beta_2) + \exp (x_i \beta_3)} \end{aligned} \]

The ratio shows us IIA

  • Consider the ratio of these two probabilities

\[ \frac{p_{i2}}{p_{i3}} = \frac{ \exp (x_i \beta_2)}{ \exp (x_i \beta_3)} \]

  • It does not depend at all on the baseline category
  • This is IIA, it is a model assumption

prob.bel <- NULL
prob.ab <- NULL

for (i in 1:100){
  df.eval.loop <- df.eval |>

    mutate(y.ihs = ihseval[i])

# But, using it for confidence intervals, here
# There, are other options, including packages

prob.bel <- NULL
prob.ab <- NULL

for (i in 1:100){
  df.eval.loop <- df.eval |> 
    dplyr::select(-y.ihs) |>
    mutate(y.ihs = ihseval[i])
  prd <- predict(mlogit, newdata = df.eval.loop, type = "probs") 
  prob.bel <- rbind(prob.bel,mean(prd[,1]))
  prob.ab <- rbind(prob.ab, mean(prd[,2]))
}

# Have only calculated probabilities
# Now, need to 'difference' the data to get margins
# Again, not meant to be perfect
margins.below <- prob.bel[-1] - prob.bel[-100]
margins.above <- prob.ab[-1] - prob.ab[-100]

ihs.ave <- (ihseval[-1] + ihseval[-100])/2

plot(c(-0.005,0.005),type = "n",
     xlab = "Range of income",
     ylab = "Change in probability")
lines(ihs.ave,margins.below,lty = 1, col ="red", lwd = 2)
lines(ihs.ave,margins.above, lty = 2, col = "blue", lwd = 2)
legend("top",
       c("Below","Above"),
       col = c("red","blue"),
       lty = c(1,2))
