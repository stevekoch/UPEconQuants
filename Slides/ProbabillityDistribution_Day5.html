Probability and Sets

Some Concepts and Terms

  1. Let \(A\) be an event, and \(B\) be another event

  2. With respect to any two events there are four possibilities

    1. \(A\) and \(B\): they both occur
    2. ~\(A\) and \(B\): \(A\) does not occur, \(B\) does
    3. \(A\) and ~\(B\): \(A\) occurs, but \(B\) does not
    4. ~\(A\) and ~\(B\): they both do not occur

  3. Draw a Venn Diagram (in class)

Some Additional Concepts and Maths

  1. Universe \(\mathbf{U}\): everything possible

  2. Thus,

    1. \(A + \sim A = U\)
    2. \(B + \sim B = U\)
    3. \(A + \sim A = B + \sim B\)
    4. \(A = B + \sim B - \sim A\)
    5. \(B = A + \sim A - \sim B\)

Yes, More Concepts

  1. Union: \(A \cup B\)
    1. All possible events where \(A\) or \(B\) occur
    2. A set containing every element from \(A\) and every element from \(B\)
    3. It includes any ‘overlap’
  2. Intersection: \(A \cap B\)
    1. Set of events that occur together
    2. Any element has to have been in \(A\) and \(B\)
    3. It is only the ‘overlap’

More Concepts and Maths I

  1. Two components to identify A…
    1. \(A = A \cap B + A \cap \sim B\)
    2. The part that intersects with B
    3. The part that intersects with ~B

More Concepts and Maths II

  1. Three objects to identify \(A \cap B\)
    1. \(A \cap B = A \cup B - A \cup \sim B + \sim A \cup B\)
    2. The set of \(A\) outside of \(B\)
    3. The set of \(B\) outside of \(A\)
    4. The joint set
    5. I am not entirely happy with this one…
  2. Look at Venn Diagrams for each (in class)

An Example Using These Concepts

Eskom has struggled to produce enough electricity for some time

  1. If this does not improve, the electricity ministered will likely be reshuffled
  2. If it does improve, the minister will likely stay
  3. Define \(A\) to be about electricity production improvements (less loadshedding)
  4. Define \(B\) as the minister staying put
  5. Assume: \(Pr(A) = 0.3\), \(Pr(B) = 0.6\) and \(Pr(A,B) = 0.2\)

Finding Some Results 1

  1. What is the probability of a reshuffle and electricity improving?
    1. This is \(Pr(A,\sim B)\)
    2. \(A = A \cup B + A \cup \sim B\)
    3. \(A \cup \sim B = A - A \cup B\)
    4. \(Pr(A \cup \sim B) = Pr(A) - Pr(A \cup B)\)
    5. \(Pr(A \cup \sim B)\) = 0.3 - 0.2 = 0.1
  2. Why did we go from \(A \cup \sim B\) to \(Pr(A \cup \sim B)\)?

Finding Some Results 2

  1. What is the probability of minister stays and electricity improving?
    1. This is \(Pr(A,B)\)
    2. Given as 0.2

Finding Some Results 3

  1. What is probability of minister staying and electricity not improving
    1. This is \(Pr(B,\sim A)\)
    2. \(B = B \cup A + B \cup \sim A\)
    3. \(B \cup \sim A = B - B \cup A\)
    4. \(Pr(B \cup \sim A) = Pr(B) - Pr(A \cup B)\)
    5. \(Pr(B \cup \sim A)\) = 0.6 - 0.2 = 0.4

Finding Some Results 4

  1. What is probability of minister reshuffle and electric not improving
    1. This is \(Pr(\sim A, \sim B)\)
    2. We know \(Pr(U) = 1 = Pr(A,B) + Pr(\sim A,B) + Pr(A, \sim B) + Pr(\sim A, \sim B)\)
    3. Law of total probability, four possible combinations, only
    4. \(Pr(\sim A, \sim B) = 1 - (Pr(A,B) + Pr(\sim A,B) + Pr(A,\sim B))\)
    5. \(Pr(\sim A, \sim B)\) = 1- .2 - .4 - 1. = 0.3

Finding Some Results 5

  1. Another use for total probability
    1. If \(Pr(A) = 0.3\), then \(Pr(\sim A) = 0.7\)
    2. If \(Pr(B) = 0.6\), then \(Pr(\sim B) = 0.4\)

Conditionality

Probability and Sets

  1. Sets tell us something about shares

  2. For example,

    1. The share of the universe made up by \(A\) is \(Pr(A)\)
    2. Technically: \(Pr(A) = A/\mathbf{U}\)

Probability and Sets II

  1. A different example
    1. The share of \(A\) is covered by the subset \(A \cap B\)?
    2. \((A \cap B)/A = .2/.3 = 2/3\)
    3. But, that is

\[ Pr(B|A) = \frac{Pr(A,B)}{Pr(A)} = .2/.3 = .67 \]

  1. In a world where \(A\) has occured, what is the probability \(B\) has also occured?

A Conditional Probability

  1. We can also calculate the reverse

\[ Pr(A|B) = \frac{Pr(B,A)}{Pr(B)} = .2/.6 = .33 \]

  1. These are conditional probabilities!!

  2. Conditioning (direction) mostly matters

More Conditionals I

  1. One for the complement of \(A\)

\[ Pr(B|\sim A) = \frac{Pr(\sim A,B)}{Pr(\sim A)} = 0.4 / 0.7 = 0.57 \]

  1. Another for the complement of B

\[ Pr(A|\sim B) = \frac{Pr(\sim B,A)}{Pr(\sim B)} = 0.1 / 0.4 = 0.25 \]

More Conditionals II

  1. Another for both complements

\[ Pr(\sim A|\sim B) = \frac{Pr(\sim B,\sim A)}{Pr(\sim B)} = 0.3 / 0.4 = 0.75 \]

  1. Please, double-check!!

Conditional Probabilities to Joint Probabilities I

  1. We already have \(Pr(A|B)\) and \(Pr(B|A)\)

  2. Note, both contain \(Pr(A,B)=Pr(B,A)\)

  3. Joint probability comes from simple manipulation

    1. \(Pr(A,B) = Pr(A|B)Pr(B)\)
    2. \(Pr(B,A) = Pr(B|A)Pr(A)\)
    3. So, \(Pr(A|B)Pr(B) = Pr(B|A)Pr(A)\)

  4. Thus, Naïve Bayes Rule

\[ Pr(A|B) = \frac{Pr(B|A)Pr(A)}{Pr(B)} \]

Additional Maths

  1. Recall
    1. \(A = A \cup B + A \cup \sim B\)
    2. Similarly, \(B = B \cup A + B \cup \sim A\)
  2. So, we can substitute out Pr(A) and Pr(B)

\[ \begin{aligned} Pr(A|B) &= \frac{Pr(B|A)Pr(A)}{Pr(B)} \\ &= \frac{Pr(B|A)Pr(A)}{Pr(B \cup A) + Pr(B \cup \sim A)}\\ &= \frac{Pr(B|A)Pr(A)}{Pr(B,A) + Pr(B,\sim A)} \\ &= \frac{Pr(B|A)Pr(A)}{Pr(B|A)Pr(A) + Pr(B|\sim A)Pr(\sim A)} \end{aligned} \]

Bayesian Decomposition

  1. From previous…

\[ Pr(A|B) = \frac{Pr(B|A)Pr(A)}{Pr(B|A)Pr(A) + Pr(B|\sim A)Pr(\sim A)} \]

  1. Likewise…

\[ Pr(B|A) = \frac{Pr(A|B)Pr(B)}{Pr(A|B)Pr(B) + Pr(A|\sim B)Pr(\sim B)} \]

Back to Eskom Example

  1. What is probability that electricity levels improved, given that minister stayed?

  2. We have all the necessary values from above.

\[ \begin{aligned} Pr(A|B) &= \frac{Pr(B|A)Pr(A)}{Pr(B|A)Pr(A) + Pr(B|\sim A)Pr(\sim A)} \\ &= \frac{0.2/0.3 \times 0.3}{0.2/0.3 \times 0.3 + 0.4/0.7 \times 0.7} \\ &= \frac{0.2}{0.2 + 0.4} \\ &= 0.2/0.6 = 1/3 \end{aligned} \]

  1. Clearly, you could ask more than a few other similar questions

Bayes and Monty Hall

A game show

  1. Choose a door from three \((D_1, D_2, D_3)\)
  2. One has a great prize, not known
  3. Two have “bad prizes”, not known
  4. Monty shows you a different door
    1. One not chosen
    2. Prizes not shuffled
  5. Door shown is never a “great prize”
  6. You can reconsider your door
  7. Question: Should you?

Further setup

  1. Without loss of generality

    1. Assume your door choice was \(D_1\)
    2. Assume Monty showed \(D_3\)

  2. Initially, \(Pr(D_i) = 1/3\) \(\forall\) \(i\)

  3. What is effect of learning about one door?

  4. In other words, should you take \(D_2\), rather than \(D_1\)?

Answering 1

  1. Initial information: \(Pr(D_1) = 1/3\), \(Pr(\sim D_1) = 2/3\)

    1. In Bayes, we often call these priors
    2. Prior information or probabilities

  2. New information: \(Pr(A_3) = 0\)

  3. So, \(Pr(D_1) = 1/3\) and \(Pr(\sim D_1) = 2/3\)

  4. BUT, \(\sim D_1\) is now just \(D_2\)

  5. Switch!

Answering with Bayes’ Law: Door 1

\[ Pr(D_1|D_3) = \frac{Pr(D_3|D_1)Pr(D_1)}{Pr(D_3)} \]

What is \(Pr(D_3|D_1)\)?

  • Recall, \(D_1\) chosen.
  • There are only two doors to be revealed: 2 or 3.
  • \(Pr(D_3|D_1) = 1/2\)
  • So, \(Pr(D_3|D_1)Pr(D_1) = 1/2 \times 1/3 = 1/6\)

Answering with Bayes’ Law: Door 2

\[ Pr(D_2|D_3) = \frac{Pr(D_3|D_2)Pr(D_2)}{Pr(D_3)} \]

What is \(Pr(D_3|D_2)\)?

  • There are only two doors to be revealed: 2 or 3.
  • After \(D_3\) is revealed, prize is not in \(D_3\)
  • \(Pr(D_3|D_2) = 1\)
  • So, \(Pr(D_3|D_2)Pr(D_2) = 1 \times 1/3 = 1/3\)

Put together

Also,

\[ \begin{aligned} Pr(D_3) = & Pr(D_3|D_1)Pr(D_1) + Pr(D_3|D_2)Pr(D_2) \\ =& 1/6 +1/3 = 1/2 \end{aligned} \]

So, for Door 1 and 2

\[ \begin{aligned} Pr(D_1|D_3) &= \frac{Pr(D_3|D_1)Pr(D_1)}{Pr(D_3)} = \frac{1/6}{1/2} = 1/3 \\ Pr(D_2|D_3) &= \frac{Pr(D_3|D_2)Pr(D_2)}{Pr(D_3)} = \frac{1/3}{1/2} = 2/3 \end{aligned} \]

Bayes Decomp 1

  1. Bayes Decomp

\[ \begin{aligned} Pr(D_{1,2}|D_3) =& \frac{Pr(D_3|D_{1,2})Pr(D_{1,2})}{Z} \\ Z=& Pr(D_3|D_1)Pr(D_1) + Pr(D_3|D_2)Pr(D_2) \\ +& Pr(D_3|D_3)Pr(D_3) \end{aligned} \]

  1. What do we mean by \(Pr(D_3|D_3)\)?
    1. Probabililty door 3, which is opened, has a great prize
    2. Not so great, we would think? 0?

Bayes Decomp 2

  1. How about \(Pr(D_3|D_1)\)?
    1. 1 is the door picked, so will not be revealed
    2. After reveal, only two doors left
    3. revealed a door with no prize
    4. Thus, the sample space is two doors
    5. probability is now 1/2

Bayes Decomp 3

  1. How about \(Pr(D_3|D_2)\)?
    1. Door 1 chosen, can’t open
    2. Probability of opening \(D_3\) if prize behind \(D_2\)?
    3. That is one - it is not saying the prize is behind \(D_2\)!!

Bayes Decomp 4

  1. What is \(Pr(D_1|D_3)\)?
    1. It is not affected by conditioning?
    2. Does that make sense?

\[ \begin{aligned} Pr(D_1|D_3) &= \frac{Pr(D_3|D_1)Pr(D_1)}{Z} \\ &= \frac{1/2 \times 1/3}{1/2 \times 1/3 + 1 \times 1/3 + 0 \times 1/3} \\ &= 1/3 \end{aligned} \]

Bayes Decomp 5

  1. What is \(Pr(D_3|D_3)\)?
    1. It is 0, as implied
    2. Does that make sense?

Bayes Decomp 6

  1. What is \(Pr(D_2|D_3)\)
    1. It is affected by conditioning
    2. Can also figure out via total probability

\[ \begin{aligned} Pr(D_2|D_3) &= \frac{Pr(D_3|D_2)Pr(D_2)}{Z} \\ &= \frac{1 \times 1/3}{1/2 \times 1/3 + 1 \times 1/3 + 0 \times 1/3} \\ &= 2/3 \end{aligned} \]

Beginnings of Estimation

What do we do with econometrics/statistics?

  • We are often interested in a population measure
    • Descriptive statistics: mean, median, variance
    • Economic parameters: elasticity, return
  • We (essentially) never have all the data
  • We estimate from a sample
  • We hope:
    • Sample ‘matches’ the relevant population
    • It is repeatable, can be recreated
    • Estimator is appropriate, given the sample

Sampling

  • Focuses on
    • ‘Matching’ the population
    • Being repeatable
    • So, clear detail and description
  • There are births, deaths, migrations
  • Not exactly random events
  • Samples are likely off (by a bit)

Modelling

  • Economic theory sits behind most questions
    • It should suggest outcomes/inputs
    • It is a relationship
    • Often presumed causal
  • The theory link has weakened over time
    • Data is cheaper to access
    • Computing time is cheaper
    • Easier to ‘fish’ for relationships

Data mining and machine learning

  • Used to be a ‘dirty’ word
  • Becoming more acceptable
  • However, it rarely offers the ‘parameters’ we seek in economics

Sample mean and variance

\[ \overline{x} = \sum_{i=1}^N \frac{x_i}{N} \]

\[ s^2 = \sum_{i=1}^N \frac{(x_i - \overline{x})^2}{N-1} \]

  • Why \(N-1\)?
  • What are degrees of freedom?

Sample minimum and maximum

\[ \min = x_i : x_i \leq x_j \forall j \in N \]

\[ \max = x_i : x_i \geq x_j \forall j \in N \]

Population median

  • Consider a real-valued random variable \(X\)
  • Consider its “right-continous” distribution function

\[ F(x) = P(X \leq x) \]

  • Thus, for the median

\[ F^{-1}(0.5) = \inf \{ x:F(x) \geq 0.5 \} \]

Sample median

  • All is the same, except we need an estimate of \(F\)
  • Call that esimate \(\hat{F}\)
  • Assume \(x\) are i.i.d.
  • The empirical cumulative distribution function is an unbiased estimate of the CDF

\[ \hat{F}_n(x) = \frac{1}{n} \sum_{i=1}^n \mathbf{1}(x_i \leq x) \]

Histograms

  • We jumped ahead a bit to use distributions
  • What is a histogram?
  • It is a frequency count across bins
    • Bins must be mutually exclusive
    • Typically, either left or right ‘inclusive’

\[ c_b = \sum_{b=1}^B m_{ib} \]

Formalising the pieces

  • Examples
    • \(m_{ib} = \mathbf{1}(L_b \leq x_i < H_b) \forall i \in N\)
    • \(m_{ib} = \mathbf{1}(L_b < x_i \leq H_b) \forall i \in N\)
  • Bin boundaries
    • \(L\) is a lower boundary value
    • \(H\) is a higher boundary
    • Importantly, \(L_1 \leq \min (X)\) and \(H_B \geq \max (X)\)

Other ideas

  • One can define bins as equally spaced
  • \(h\) is a bandwidth that is the length of each bin
  • Examples:
    • \(k = \left[ (\max(X) - \min(X))/h \right]\)
    • \(k = log_2 n + 1\)
  • The choices for \(h\) and \(k\) are numerous
  • Some are ‘data-driven’

Densities

  • With discrete data
    • Histograms are densities
    • But should be per outcome (not grouped)
    • And, needs to be in probabilities, not frequencies
  • With continuous data
    • Histogram is a group density
    • Continuous data would have too many bars
    • So, need a smooth histogram

Estimating a Continuous Density

  • What is a density?
  • Mathematically, it is a well-defined object

\[ f(x) = \frac{d}{dx}F(x) \]

  • We measure \(F\) via the empirical CDF, from above

\[ \hat{F}_n(x) = \frac{1}{n} \sum_{i=1}^n \mathbf{1}(x_i \leq x) \]

A version of a derivative definition

\[ \hat{f}(x) = \frac{F(x+h)-F(x-h)}{2h} \]

  • This is essentiall a histogram
  • Data intervals are \(2h = h - (-h)\)
  • This PDF estimate is intuitive
    • The average number of observations in an interval
    • Normalized by the length of the interval

Substitute the Empirical CDF into the PDF

  • Unfortunately, histograms are not smooth

\[ \hat{f}(x) = \frac{1}{2nh} \sum_{i=1}^n \left( \mathbf{1} (x_i \leq x+h) - \mathbf{1} (x_i \leq x-h) \right) \]

  • So, some manipulation

\[ \begin{split} x_i \leq x+ h &\leftrightarrow x_i - x &\leq h \\ \frac{x_i-x}{h} &\leq 1 \end{split} \]

Evaluating the weak inequality

  • Rewrite with absolute values

\[ \left\lvert \frac{x_i - x}{h} \right\rvert \leq 1 \]

  • Break it apart

\[ \begin{split} \frac{x_i -x}{h} &\leq 1 \\ \frac{x_i - x}{h} & \geq -1 \end{split} \]

Make a naïve assumption

  • This matches the expressions above

\[ k\left(\frac{x_i-x}{h} \right) = \begin{cases} \frac{1}{2} & \text{if } | \frac{x_i-x}{h} | \leq 1 \\ 0 & \text{otherwise} \end{cases} \]

  • Substituting from above

\[ \begin{split} \hat{f}(x) &= \frac{1}{nh} \sum_{i=1}^n \left( -\frac{1}{2} \leq \frac{x_i - x} {h} \leq \frac{1}{2} \right) \\ &= \frac{1}{nh} \sum_{i=1}^n k\left(\frac{x_i-x}{h} \right) \end{split} \]

The naïve kernel

  • See the following equation

\[ \hat{f}(x) = \frac{1}{nh} \sum_{i=1}^n k\left(\frac{x_i-x}{h} \right) \]

  • We can use other kernels
    • Gaussian
    • Epanechnikov
    • Triangular

Underlying required properties

  • Kernels should satisfy these 3 properties
  • Symmetry is not really required, but easier to use

\[ \begin{split} \int k(v)dv &= 1 \qquad \text{a proper probability} \\ k(v) &= k(-v) \qquad \text{symmetric} \\ \int v^2 k(v)dv &= \kappa_2 >0 \qquad \text{an existant variance} \end{split} \]

  • The data is fairly normal, and the estimate fits the true
  • Observations at 1000, so not too surprising
options(np.messages=FALSE)

###### nonparametric regression ####
bw <- npudensbw(~ln_income,
                data = df,
                bwmethod = "cv.ls",
                ckerorder=4,
                nmulti=5)

out <- plot(bws=bw,
            neval=75,
            perspective=FALSE,
            plot.errors.method="bootstrap",
            plot.errors.boot.num=399,
            plot.errors.type="quantiles",
            plot.behavior="data")

np.fit <- out$d1$dens
np.se <- (out$d1$derr)
np.Ul <- np.fit + np.se[,2]
np.Ll <- np.fit + np.se[,1]
xeval <- out$d1$eval$ln_income

xseq <- seq(min(ln_income),max(ln_income),0.25)
xnorm <- dnorm(xseq,mean(ln_income),sd(ln_income))

hist(ln_income,freq=FALSE,
     ylim=c(0,0.25),
     col="pink",border="red",
     xlab = "The Support of log income",
     ylab = "Density of log income",
     main="Histogram and Density of log income")
lines(np.fit~xeval,col=2,lwd=2,lty=1) 
  lines(np.Ul~xeval,col=2,lwd=1,lty=2) 
  lines(np.Ll~xeval,col=2,lwd=1,lty=2) 
lines(xseq,xnorm,col=4,lwd=2,lty=3)  
legend("topright",
       lwd=c(2,1,2),
       lty=c(1,2,3),
       col=c(2,2,4),
       #pch=c(NA,NA,16),
       #pt.cex=1,
       c("Nonparametric Density","Nonparametric 95\\% CI","Normal Density"))
